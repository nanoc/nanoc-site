nanoc 3.2


This page is an overview of the development activity on nanoc 3.2. Also see the sub-pages:

 * improved performance
 * architectural changes
 * the compilation process


Compatibility


nanoc 3.2 will remain compatible with nanoc 3.1. Sites built with nanoc 3.1 and 3.0 should be compileable without issues on nanoc 3.2.


Requirements


 must (requirement must be fulfilled)::
   * #70: Improved file outdatedness checks
   * #100: Autocompiler that recompiles on change
 should (requirement should be fulfilled but is not a show stopper)::
   * #71: Improved layout dependency tracking
   * #101: Storing already compiled content
   * #102: Improved dependency tracking
   * #103: Recording actions necessary for compiling an item
 nice to have (requirement is purely optional)::
   * #65: Better UI for create_site command
   * #92: Allow src/ as well as content/
   * #104: Storing compiled snapshots
   * #108: Handling unmet dependencies more efficiently
   * #111: Split base in three sub-modules
   * #112: Don’t make mtime handling special
   * #116: Auto-load data
   * #125: One-time extraction of metadata
 future (this might be a good idea for a later release but should not influence the current version)::
   * #98: Do not depend on current working directory
   * #75: Make nanoc look in parent directories
   * #109: Parallellize the compilation process
