nanoc 3.1 Brainstorming


This page collects ideas and discussions about nanoc 3.1. Also see the sub-pages: binary assets, better gem and executable names, logo.


Compatibility


nanoc 3.1 will remain compatible with nanoc 3.0. Sites built with nanoc 3.0 should be compileable without issues on nanoc 3.1.


Requirements


 must (requirement must be fulfilled)::
   * #32: The output directory should behave like a build directory
   * #66: Support for binary items
 should (requirement should be fulfilled but is not a show stopper)::
   * #76: Content and metadata files should be optional
   * #82: Give gem and executable better names
   * #91: Hide item rep complexity
 nice to have (requirement is purely optional)::
   * #68: Link validation task
   * #84: Logo
   * #88: Syntax coloring filter
   * #79: @-vars for filters
   * #96: Let atom_feed take attributes from params and configuration
   * #97: Make atom_feed use updated_at attribute
 future (this might be a good idea for a later release but should not influence the current version)::
   * (nothing)
